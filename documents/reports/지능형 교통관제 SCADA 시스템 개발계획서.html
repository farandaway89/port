시나리오 예시 - UC02

적응형 신호제어
1. 교차로 A에서 북-남 방향 교통량 급증 감지
2. 3초마다 센서 데이터 수집
3. 교통량 임계값(50대/분) 초과 확인
4. 해당 방향 녹색신호 20% 연장
5. 인접 교차로 연동 신호 조정
6. "교통흐름 최적화 완료" 알림 전송
7. 제어이력 로그 기록 저장

주요 관계 설명

관계 정의:
• USERS ↔ NOTIFICATIONS (1:N)
• USERS ↔ SIGNAL_LOGS (1:N)
• INTERSECTIONS ↔ TRAFFIC_DATA (1:N)
• INTERSECTIONS ↔ SIGNAL_LIGHTS (1:N)
• SIGNAL_LIGHTS ↔ SIGNAL_LOGS (1:N)
• TRAFFIC_PATTERNS (독립 참조 테이블)

인덱스 설계

• traffic_data: (timestamp, intersection_id)
• signal_logs: (timestamp, signal_id)
• notifications: (user_id, timestamp)
• intersections: (latitude, longitude)
• users: username (unique)

데이터 보존 정책

보존 기간:
• traffic_data: 1년 → 월단위 집계
• signal_logs: 6개월 → 중요 이벤트만
• notifications: 3개월 → 삭제
• 기타 마스터: 영구 보존

성능 최적화

• 파티셔닝: 월별 테이블 분할
• 압축: 과거 데이터 압축 저장
• 캐싱: 실시간 데이터 메모리 캐시
• 배치: 대용량 데이터 일괄 처리
• 복제: 읽기 전용 복제본 운영
