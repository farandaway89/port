시나리오 예시 - UC01

부동산 검색 및 필터링
1. 사용자가 검색어 입력 (주소, 지역명, 아파트명)
2. 가격 범위, 방 개수, 욕실 개수 등 필터 설정
3. 부동산 유형, 상태, 신뢰도 점수 필터 적용
4. 정렬 옵션 선택 (가격, 날짜, 신뢰도)
5. 검색 결과를 카드 형태로 표시
6. 실시간 필터 변경 시 즉시 결과 업데이트
7. 검색 이력 자동 저장

주요 관계 설명

관계 정의:
• PROPERTIES ↔ FAVORITES (1:N)
• SEARCH_HISTORY (독립 테이블)
• 외래키 제약조건으로 데이터 무결성 보장

인덱스 설계

• properties: (price, beds, baths, trust_score)
• properties: (address) - 텍스트 검색용
• properties: (latitude, longitude) - 지도 검색용
• favorites: (user_id, property_id)
• search_history: (created_at)

데이터 보존 정책

보존 기간:
• properties: 영구 보존
• favorites: 사용자별 영구 보존
• search_history: 1년 → 자동 삭제
• 백업: 일일 자동 백업
• 아카이빙: 월별 데이터 압축

성능 최적화

• 인덱스: 검색 성능 최적화
• 파티셔닝: 날짜별 테이블 분할
• 캐싱: 자주 검색되는 데이터 메모리 캐시
• 쿼리 최적화: Prepared Statement 사용
• 연결 풀링: 데이터베이스 연결 효율적 관리
