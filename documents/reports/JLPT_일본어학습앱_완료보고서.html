3.2 데이터베이스 설계 및 관리

SQLite 데이터베이스 구조

DatabaseManager.js - 3개 테이블 설계:

words (단어 테이블)

CREATE TABLE words ( id INTEGER PRIMARY KEY, kanji TEXT, reading TEXT NOT NULL, meaning TEXT NOT NULL, english TEXT, jlpt_level INTEGER NOT NULL, part_of_speech TEXT, example TEXT, example_meaning TEXT );

저장 데이터: 21,300+ 단어

learning_progress
(학습 진도)

CREATE TABLE learning_progress ( word_id INTEGER PRIMARY KEY, learned_count INTEGER DEFAULT 0, correct_count INTEGER DEFAULT 0, last_studied DATETIME, next_review DATETIME, difficulty INTEGER DEFAULT 1, FOREIGN KEY (word_id) REFERENCES words(id) );

기능: 복습 알고리즘 기반 학습 관리

test_records
(테스트 기록)

CREATE TABLE test_records ( id INTEGER PRIMARY KEY, jlpt_level INTEGER NOT NULL, total_questions INTEGER, correct_answers INTEGER, test_date DATETIME, duration INTEGER );

기능: 테스트 이력 및 통계 관리

간격 반복 학습 알고리즘 (Spaced Repetition):

// 난이도 기반 복습 간격 계산 calculateNextReview(difficulty) { const intervals = [1, 3, 7, 14, 30]; // 일 단위 const intervalIndex = Math.min(difficulty - 1, intervals.length - 1); const days = intervals[intervalIndex]; const nextReview = new Date(); nextReview.setDate(nextReview.getDate() + days); return nextReview.toISOString(); } // 학습 진도 업데이트 로직 async updateLearningProgress(wordId, isCorrect) { const current = await db.getFirstAsync( 'SELECT * FROM learning_progress WHERE word_id = ?', [wordId] ); let difficulty = 1; if (current) { // 정답: 난이도 감소, 오답: 난이도 증가 difficulty = isCorrect ? Math.max(1, current.difficulty - 1) : Math.min(5, current.difficulty + 1); } const nextReview = this.calculateNextReview(difficulty); await db.runAsync(` INSERT OR REPLACE INTO learning_progress (word_id, learned_count, correct_count, last_studied, next_review, difficulty) VALUES (?, ?, ?, ?, ?, ?) `, [wordId, learnedCount, correctCount, now, nextReview, difficulty]); }

알고리즘 특징:

  • 1일 후: 새로 학습한 단어 (difficulty = 1)
  • 3일 후: 한 번 복습한 단어 (difficulty = 2)
  • 7일 후: 두 번 복습한 단어 (difficulty = 3)
  • 14일 후: 세 번 복습한 단어 (difficulty = 4)
  • 30일 후: 완전히 숙지한 단어 (difficulty = 5)

화면 설명

1번 화면 (개발 디버깅): 개발 과정에서 발생한 데이터베이스 오류를 해결하는 과정. React Native의 실시간 디버깅 기능을 통해 문제를 추적하고 수정했습니다.

2번 화면 (레벨 선택): JLPT N5(기초)부터 N1(고급)까지 5단계 레벨을 시각적으로 구분하여 표시. 각 레벨별 단어 수를 표시하여 사용자가 학습 범위를 파악할 수 있습니다.

3번 화면 (홈 화면): 학습 진도율과 통계를 한눈에 확인할 수 있는 대시보드. 학습한 단어 수, 연속 학습일, 진도율 프로그레스 바를 표시합니다.

📱 주요 화면 기능 요약

  • 홈 화면: 학습 통계, 진도율, 빠른 시작 버튼
  • 레벨 선택: JLPT N5~N1 레벨별 단어 수 표시
  • 플래시카드: 카드 플립, 음성 재생, 진도 표시
  • 단어 테스트: 객관식 문제, 정답률 실시간 표시
  • 테스트 결과: 점수, 정답/오답 분석
  • 통계 화면: 학습 이력, 레벨별 진도, 그래프

  • 데이터 모델링 및 정규화
  • 로컬 스토리지 최적화
7.4 향후 개선 방향

단기 개선 계획

  • iOS 버전 개발: React Native의 크로스 플랫폼 장점을 활용하여 iOS 앱 출시
  • 오프라인 음성 파일: 네이티브 음성 파일 포함하여 인터넷 없이도 발음 재생
  • 단어장 기능: 사용자 정의 단어장 생성 및 관리
  • 소셜 기능: 친구와 학습 진도 공유 및 경쟁

  • AI 추천 시스템: 사용자 학습 패턴 분석하여 개인화된 단어 추천
  • 게임화 요소: 학습 배지, 레벨업 시스템, 리더보드
  • 문법 학습 확장: 단어뿐만 아니라 JLPT 문법 학습 콘텐츠 추가
  • 클라우드 동기화: 여러 기기 간 학습 진도 동기화
