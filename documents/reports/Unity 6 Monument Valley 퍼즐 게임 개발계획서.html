📋 1. 프로젝트 개요

🎯 프로젝트 목표

주 목표: Unity 6과 호환되는 Monument Valley 스타일의 3D 퍼즐 게임 구현

부 목표: 기존 스파르타 전설 프로젝트를 현대적 게임으로 전환

최종 목표: 4K 환경에서 완전히 작동하는 게임 제작

개발 환경 사양
• Unity 버전: Unity 6 (최신)
• 프로그래밍 언어: C#
• 플랫폼: Windows (4K 해상도)
• 아트 스타일: Monument Valley 미니멀리즘

⚠️ 예상 주요 도전과제

Unity 6 호환성: 기존 API 변경사항 대응 필요

Input System: Legacy vs New Input System 충돌

4K UI: 고해상도 환경 텍스트 가독성

Monument Valley 스타일: 정확한 미니멀 디자인 구현

🗓️ 2. 개발 단계별 계획

1단계: Unity 6 호환성 분석 및 수정 (2시간)

목표: 기존 스파르타 전설 프로젝트의 Unity 6 호환성 확보

  • 컴파일 에러 전체 식별 및 목록화
  • FindObjectOfType → FindFirstObjectByType 변경
  • Rigidbody.velocity → Rigidbody.linearVelocity 변경
  • 기타 deprecated API 전체 업데이트

성공 기준: 모든 스크립트 컴파일 성공

2단계: Input System 최적화 (2시간)

목표: Unity Input System 충돌 문제 완전 해결

  • Legacy Input Manager vs New Input System 충돌 분석
  • SafeInputHandler 클래스 구현으로 방어적 프로그래밍
  • try-catch 예외 처리로 안정성 확보
  • ProjectSettings.asset 직접 수정으로 Input 방식 통일

성공 기준: 키보드/마우스 입력 정상 작동

3단계: Monument Valley 캐릭터 구현 (1.5시간)

목표: Monument Valley의 미니멀한 캐릭터 디자인 구현

  • 기하학적 형태의 단순한 캐릭터 디자인
  • 파스텔 색상 팔레트 적용
  • 4K 해상도에서 명확한 시각성 확보
  • 애니메이션 없는 정적 우아함 추구

성공 기준: Monument Valley 스타일에 부합하는 캐릭터

4단계: 게임 메커니즘 구현 (1.5시간)

목표: 직관적이고 매력적인 게임플레이 구현

  • WASD 키보드 이동 시스템 구현
  • 마우스 클릭 상호작용 시스템
  • 목표 지점 도달 감지 및 성공 처리
  • 기본적인 물리 시뮬레이션 적용

성공 기준: 완전한 게임 루프 작동

5단계: 대안 설계 (구조물 퍼즐) (1시간)

목표: 캐릭터 구현 실패 시 구조물 기반 게임으로 전환

  • 회전 가능한 구조물 시스템 (L자, T자, 브릿지)
  • 물리 기반 구슬 굴리기 메커니즘
  • OnMouseDown 기반 직접 조작
  • Monument Valley 핵심 요소인 구조물 조작에 집중

성공 기준: 구조물 회전을 통한 퍼즐 해결

⚙️ 3. 기술적 구현 계획
Unity 6 API 대응
• FindFirstObjectByType 사용
• linearVelocity 속성 활용
• 최신 Physics API 적용
• Component 접근법 업데이트
Input System 전략
• Legacy Input 우선 적용
• New Input System 백업
• 방어적 예외 처리
• 다중 입력 방식 지원
UI/UX 최적화
• 4K 동적 스케일링
• GUI.matrix 활용
• 텍스트 가독성 보장
• 해상도 무관 UI

핵심 스크립트 구조

  • MonumentPuzzleGame.cs: 메인 게임 매니저
  • RotatableStructure.cs: 구조물 회전 시스템
  • GoalTrigger.cs: 목표 달성 감지
  • PuzzleCameraController.cs: 카메라 제어
  • SafeInputHandler.cs: 안전한 입력 처리

물리 시스템 설계

  • Rigidbody: 구슬 물리 시뮬레이션
  • Collider: 구조물 충돌 감지
  • PhysicMaterial: 마찰 및 반발 조정
  • Gravity: 현실적인 중력 효과
  • Constraints: 불필요한 회전 제한

⚠️ 5. 위험 요소 및 대응 방안

🔴 높은 위험도

Unity Input System 충돌

Legacy와 New Input System 동시 존재로 인한 InvalidOperationException 발생 가능성

대응방안: ProjectSettings 직접 수정, 방어적 프로그래밍, 다중 백업 시스템

🟡 중간 위험도

4K 해상도 UI 문제

고해상도에서 텍스트가 너무 작아 가독성 저하

대응방안: GUI.matrix를 이용한 동적 스케일링 구현

🟡 중간 위험도

Monument Valley 스타일 구현 실패

복잡한 캐릭터 디자인으로 미니멀 스타일 위배 가능성

대응방안: 구조물 중심 퍼즐로 완전 방향 전환

🟢 낮은 위험도

물리 시뮬레이션 불안정

Rigidbody 동작 예측 어려움 및 비현실적 움직임

대응방안: 적절한 mass, drag 값 조정 및 Constraints 활용
