1. 👥 유스케이스 (Use Case)

시스템 액터

👨‍💼 일반 투자자

소액 투자 및 2차 거래 참여

🧐 전문가 (큐레이터)

투자 상품 평가 및 추천

🏢 자산 보유자

부동산, 예술품 등 자산 유동화

⚙️ 플랫폼 관리자

시스템 운영 및 감독

주요 유스케이스

UC01: 투자 상품 조회
다양한 대체투자 상품 탐색
UC02: 크라우드펀딩 참여
소액으로 투자 상품에 투자
UC03: 2차 거래
보유한 투자증권을 다른 투자자와 거래
UC04: 포트폴리오 관리
투자 현황 및 수익률 확인

시나리오 예시 - UC02

크라우드펀딩 참여
1. 투자자가 '10억 빌딩' 상품 선택
2. 투자금액 100,000원 입력
3. 블록체인 기반 스마트 컨트랙트 확인
4. 예치금 계좌에서 투자금 이체
5. 투자금액에 해당하는 '빌딩 토큰' 100개 발급
6. 내 지갑에서 토큰 확인
7. 투자 내역 블록체인에 기록
2. 📋 요구사항 명세서

기능적 요구사항

ID 요구사항 우선순위
FR01 사용자 회원가입 및 인증 (KYC) 높음
FR02 대체투자 상품 등록 및 관리 높음
FR03 크라우드펀딩 기반 투자 기능 높음
FR04 블록체인 기반 토큰 발행 높음
FR05 2차 거래 시장 (매수/매도) 중간
FR06 투자 포트폴리오 관리 중간
FR07 전문가 큐레이션 서비스 낮음

비기능적 요구사항

ID 요구사항 목표값
NFR01 가용성 (AWS) 99.9% 이상
NFR02 응답 시간 (API) 200ms 이내
NFR03 초당 트랜잭션 (TPS) 1,000 TPS 이상
NFR04 데이터 무결성 블록체인으로 보장
NFR05 보안 개인정보 암호화, 스마트 컨트랙트 감사

제약사항

• Frontend: React.js
• Backend: NestJS (Node.js)
• Database: MySQL
• Infrastructure: AWS (EC2, S3, RDS)
• Blockchain: EVM 호환 블록체인 (e.g., Ethereum, Polygon)

6. 📅 일정표 (12개월)
단계 기간 주요 작업 상세 내용 산출물
1단계 1-3개월 기획 및 코어 백엔드 설계 요구사항 구체화, 아키텍처 설계, DB 스키마, 블록체인 PoC 상세명세서, 아키텍처도, ERD, PoC 보고서
2단계 4-6개월 MVP 개발 React 기반 프론트엔드, 투자/회원가입 API, 토큰 발행 컨트랙트 MVP 버전 웹, API 문서, 스마트 컨트랙트
3단계 7-9개월 2차 거래 및 고도화 2차 거래소 구현, 전문가 큐레이션, 관리자 대시보드 개발 2차 거래소, 큐레이션 기능, 관리자 페이지
4단계 10-12개월 테스트, 감사 및 런칭 QA, 보안 감사(Certik 등), AWS 인프라 구축, 최종 배포 테스트 보고서, 감사 리포트, 운영서버, 정식 서비스

🔗 블록체인 및 보안

블록체인 적용
  • ERC-1155 기반 투자증권 토큰화
  • 스마트 컨트랙트를 통한 투자 계약 자동 체결
  • 모든 거래 내역 블록체인에 기록 (투명성)
  • 분산원장을 통한 데이터 무결성 확보
보안 시스템
  • KYC/AML 통한 사용자 인증
  • 개인정보 및 민감 데이터 암호화
  • 2FA (2-Factor Authentication) 지원
  • 정기적인 보안 취약점 점검
