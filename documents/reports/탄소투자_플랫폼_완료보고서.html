⚖️ 프로젝트 혼재 결과

본 프로젝트는 계획된 기능 중 약 85%를 성공적으로 구현했으나, 2차 거래 시장의 고급 기능(시장가 주문, 파생상품)과 전문가 큐레이션의 AI 추천 시스템은 시간 제약 및 기술적 복잡성으로 인해 부분 구현 또는 프로토타입 수준에 머물렀습니다. 핵심 기능인 크라우드펀딩을 통한 소액 투자와 블록체인 기반 토큰 발행은 안정적으로 작동하지만, 완전한 탈중앙화 거래소(DEX) 수준의 유동성 제공에는 한계가 있습니다. 전반적으로 상용화 초기 버전(MVP)으로는 충분한 수준의 시스템을 구축했으나, 완전한 기능 확장을 위해서는 추가 개발이 필요합니다.

✅ 유스케이스 (Use Case) - 구현 현황

핵심 유스케이스 구현 현황 (혼재 결과)

✓ UC01: 투자 상품 조회
부동산, 예술품, 와인 등 다양한 상품을 카테고리별로 조회하고 상세 정보를 확인하는 기능 완전 구현
✓ UC02: 크라우드펀딩 참여
소액으로 투자 상품에 펀딩하고, 투자금액에 비례하여 블록체인 기반 증권형 토큰(STO)을 발급받는 기능 완전 구현
△ UC03: 2차 거래
지정가 주문을 통한 사용자 간 토큰 거래는 가능하나, 시장가 주문 및 자동 매칭 시스템은 부분적으로만 구현됨
✓ UC04: 포트폴리오 관리
보유 자산, 총 투자액, 평가수익률 등 개인 포트폴리오를 시각적으로 확인하는 기능 완전 구현
✗ UC05: AI 기반 큐레이션
전문가가 수동으로 상품을 추천하는 기능만 구현, AI 기반 개인 맞춤 추천 시스템은 프로토타입 개발 후 중단

부분 성공 시나리오 - UC03 (2차 거래)

제한적 성공
1. 사용자는 보유한 토큰을 지정된 가격과 수량으로 판매 주문(매도 호가)을 등록할 수 있습니다.
2. 다른 사용자는 등록된 매도 호가를 보고 해당 가격과 수량으로 구매할 수 있습니다. (지정가 거래)
3. 거래가 체결되면 스마트 컨트랙트를 통해 안전하게 토큰과 자산이 교환됩니다.
4. 그러나, 실시간으로 변하는 가격에 즉시 체결되는 시장가 주문 기능은 구현되지 않았습니다.
5. 복잡한 조건의 주문(Stop-loss 등)은 지원하지 않습니다.

💡 주요 배움과 한계 인식

이번 대체투자 플랫폼 프로젝트는 웹2 기술과 웹3(블록체인) 기술을 융합하는 과정의 복잡성을 깊이 체감한 경험이었습니다. 핵심 투자 기능은 성공적으로 구현했지만, 실시간성과 탈중앙성이 중요한 2차 거래소 개발에서는 명확한 한계를 보였습니다.

🔬 기술적 도전과 한계

블록체인 연동의 복잡성

한계: 스마트 컨트랙트의 비동기적 특성과 최종성(Finality)을 기다리는 로직 처리가 웹 애플리케이션의 즉각적인 반응성과 충돌하는 부분이 많았습니다. 가스비 문제 또한 사용자 경험을 해치는 주요 요인이었습니다.

학습: 블록체인 트랜잭션의 상태(Pending, Success, Failed)를 사용자에게 명확히 피드백하는 UI/UX 설계의 중요성을 깨달았습니다.

📈 의외의 성공과 성취

견고한 API 및 데이터 모델

성공: NestJS와 TypeORM을 사용한 백엔드 API는 매우 안정적이고 확장 가능하게 설계되었습니다. 특히 투자 상품과 사용자 자산을 관리하는 데이터 모델은 성공적으로 구축되어 핵심 기능의 안정성을 높였습니다.

발견: 복잡한 시스템일수록, 기본이 되는 API와 데이터베이스 설계의 중요성이 더욱 크다는 것을 확인했습니다.
