플래시카드 기반 JLPT 단어 학습 어플리케이션 개발 계획서

React Native 크로스 플랫폼 모바일 앱

프로젝트 기간: 2025/09/23 ~ 2025/09/27 (5일간) | 개인 프로젝트 | 포트폴리오용
1. 프로젝트 개요

목적 및 목표

주요 목적
• JLPT 단어 학습을 위한 효과적인 모바일 앱 개발
• 개인 개발 역량 증명용 포트폴리오 작품
• 오픈소스 데이터 활용한 무료 학습 앱 제공
• React Native 기술 스택 숙련도 향상

기술적 특징

무료 개발
• 오픈소스 데이터 활용 (JMdict, Anki 덱)
• React Native 무료 프레임워크
• SQLite 로컬 데이터베이스
• APK 직접 배포 (스토어 등록비 절약)
2. 기술 스택

프론트엔드

React Native 0.72+
• JavaScript/TypeScript
• React Navigation 6
• React Native Paper (UI)
• Async Storage
• Vector Icons

데이터베이스

SQLite (로컬)
• react-native-sqlite-storage
• 오프라인 사용 가능
• 빠른 검색 성능
• 용량 효율적

개발 도구

개발 환경
• VS Code
• Android Studio
• Git
• npm/yarn

데이터 소스

JMdict (일본어-영어 사전)
• 약 180,000개 단어
• Creative Commons 라이선스
• 한자, 히라가나, 카타카나 포함
• 품사 정보 제공
Anki 오픈소스 덱
• JLPT N5~N1 레벨별 정제된 데이터
• 커뮤니티 검증 완료
• 예문 및 발음 정보 포함
• 학습 최적화된 구조
3. 요구사항 명세서

기능적 요구사항

ID 요구사항 우선순위
FR01 JLPT 레벨별 단어 데이터 로드 높음
FR02 플래시카드 학습 인터페이스 높음
FR03 학습 진도 저장 및 관리 높음
FR04 단어 테스트 모드 중간
FR05 복습 알고리즘 구현 중간
FR06 통계 및 학습 분석 낮음

비기능적 요구사항

ID 요구사항 목표값
NFR01 앱 시작 시간 3초 이내
NFR02 단어 검색 속도 1초 이내
NFR03 오프라인 사용 100% 지원
NFR04 메모리 사용량 100MB 이하
NFR05 지원 안드로이드 버전 API 21+ (5.0+)

제약사항

• 안드로이드 전용 (iOS는 향후 확장)
• 오프라인 우선 설계
• 인터넷 연결 불필요
• 개인 프로젝트로 상업적 배포 없음

6. 개발 일정
일차 날짜 주요 작업 상세 내용 산출물
1일차 09/23 (토) 프로젝트 설정 및 데이터 준비 React Native 프로젝트 생성, JMdict 데이터 다운로드/가공, SQLite 스키마 설계 프로젝트 기본 구조, 단어 데이터베이스
2일차 09/24 (일) 기본 UI 컴포넌트 개발 네비게이션 설정, 메인 화면, 레벨 선택 화면, 기본 스타일링 기본 화면 구조, UI 컴포넌트
3일차 09/25 (월) 플래시카드 학습 기능 단어 카드 컴포넌트, 학습 로직, 진도 저장, 복습 알고리즘 플래시카드 학습 시스템
4일차 09/26 (화) 테스트 모드 및 통계 객관식 테스트, 점수 기록, 학습 통계, 진도 관리 테스트 시스템, 통계 화면
5일차 09/27 (수) 최종 완성 및 APK 빌드 버그 수정, UI 개선, 성능 최적화, APK 빌드 및 테스트 완성된 APK 파일, 설치 테스트
