주요 유스케이스

UC01: 실시간 수질 모니터링
8개 센서 수질 파라미터 실시간 확인
UC02: 자동 수처리 제어
수질 기준값 기반 장비 자동 제어

pH 조절제, 응집제, 소독제 자동 투입
UC04: 처리공정별 설정 관리
공정별 최적 운영조건 설정

시나리오 예시 - UC02

자동 수처리 제어
1. 서버는 MySQL DB에서 처리 기준값을 로드합니다
2. 3초마다 8개 센서 데이터 수집
3. pH 데이터가 기준값(7.0±0.5) 범위 벗어남
4. 해당 조절제 펌프 자동 가동
5. pH 정상 범위 회복 후 자동 정지
6. "수질 정상화 완료" 알림 전송
7. 제어이력 로그 기록 저장

주요 관계 설명

관계 정의:
• USERS ↔ NOTIFICATIONS (1:N)
• USERS ↔ TREATMENT_LOGS (1:N)
• SENSORS ↔ WATER_QUALITY_DATA (1:N)
• EQUIPMENT ↔ TREATMENT_LOGS (1:N)
• CHEMICAL_DOSING (독립 테이블)

인덱스 설계

• water_quality_data: (timestamp, sensor_id)
• treatment_logs: (timestamp, equipment_id)
• notifications: (user_id, timestamp)
• chemical_dosing: (timestamp, chemical_type)
• users: username (unique)

데이터 보존 정책

보존 기간:
• water_quality_data: 5년 → 월단위 집계
• treatment_logs: 2년 → 중요 이벤트만
• chemical_dosing: 3년 → 법적 요구사항
• notifications: 1년 → 삭제
• 기타 마스터: 영구 보존

성능 최적화

• 파티셔닝: 월별 테이블 분할
• 압축: 과거 데이터 압축 저장
• 캐싱: 실시간 데이터 메모리 캐시
• 배치: 대용량 데이터 일괄 처리
• 복제: 읽기 전용 복제본 운영
