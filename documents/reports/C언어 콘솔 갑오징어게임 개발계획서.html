3. 게임 라운드별 기능 설계

🌸 Round 1: 무궁화 꽃이 피었습니다

📋 게임 규칙
  • 감시자가 "무궁화 꽃이 피었습니다"를 외치며 돌고 있음
  • 말하는 중에는 움직여야 하고, 멈추면 안 됨
  • 돌았을 때 움직이면 실격
  • 성공 조건: 일정 거리 도달
⚙️ 구현 기능
  • 시간 단위 상태 변경 (움직임 허용/금지)
  • 키 입력 감지 (WASD)
  • 움직임 판단 (금지 시간 중 입력 시 탈락)
  • 성공 조건: 일정 거리 도달

🍪 Round 2: 뽑기

📋 게임 규칙
  • 주어진 모양을 도구 없이 정해진 시간 안에 오차 없이 찍어야 함
  • 실수하거나 시간 초과하면 실패
  • 도형: 원/별/우산 등
⚙️ 구현 기능
  • 도형 랜덤 선택 (원/별/우산 등)
  • 경로 추적 시스템 (좌표 기반 입력)
  • 방향키로 선을 따라가는 시스템
  • 오차 감지 및 실패 처리
  • 타이머 동작

🪨 Round 3: 비석치기

📋 게임 규칙
  • 바닥에 세운 비석(돌)을 멀리서 던져 맞추면 성공
  • 힘 조절과 각도 조절이 필요
  • 날씨별 성공률 변화
⚙️ 구현 기능
  • 게이지에 따라 "던지기 성공 확률" 조절
  • 게이지 중앙(7~8)에서만 명중률 높음
  • 날씨 시스템: 비 70%, 안개 50% 성공률
  • 캐릭터 능력치 반영

🌉 Round 4: 징검다리

📋 게임 규칙
  • 징검다리를 건너야 함
  • 두 개 중 하나는 깨짐
  • 정답 유리는 무작위로 배치
  • 제한 시간 내 다 건너야 함
⚙️ 구현 기능
  • 무작위 징검다리 배열 생성
  • 선택 입력 (왼쪽 or 오른쪽)
  • 정답/오답 판단 → 오답 시 낙하 탈락
  • 제한 시간 타이머

7. 예상 문제점 및 대응 방안
문제점 원인 영향도 대응 방안 담당자
콘솔 입출력의 한계 실시간 입력이나 화면 제어의 어려움 높음 ncurses 라이브러리 검토, 대안 방법 모색 문태영
게임 루프 설계 미숙 메인 루프 구조 설계 경험 부족 높음 단계적 구현, 모듈화된 설계 이승필
상태 관리 어려움 4개 라운드의 복잡한 상태 관리 중간 구조체 활용, 전역 변수 최소화 이주희
시간/속도 제어 어려움 일정 시간 간격 게임 진행 제어 중간 sleep 함수 활용, 타이머 구현 최연주
게임 디버깅 능력 부족 복잡한 게임 로직의 오류 추적 중간 printf 디버깅, 단위별 테스트 전체
UI/게임 요소 구성 미숙 ASCII 아트와 콘솔 UI 경험 부족 낮음 간단한 UI부터 점진적 개선 전체
