Smart Building Monitor

C#, WPF, MongoDB 기반 스마트 빌딩 에너지 모니터링 솔루션

프로젝트 개발계획서

개발 기간: 2025.11.10 - 2025.11.17 (8일)

개발자: 이승필

1. 프로젝트 개요

프로젝트 목표

실시간 센서 데이터 수집 및 시각화를 통한 스마트 빌딩 에너지 관리 시스템 개발. 30개 센서(온도, 습도, 전력, 태양광)의 데이터를 MongoDB에 저장하고, 2D 차트(OxyPlot)와 3D 시각화(HelixToolkit)를 통해 직관적인 모니터링 환경을 제공합니다.

주요 기능

실시간 데이터 모니터링
30개 센서의 실시간 데이터 수집 및 표시
2D 차트 시각화
OxyPlot을 활용한 시계열 데이터 그래프
3D 빌딩 뷰
HelixToolkit 기반 3층 건물 3D 시각화

기술 스택

C# 9.0 WPF .NET 9.0 MongoDB 7.0 OxyPlot 2.2 HelixToolkit.Wpf 2.24 CommunityToolkit.Mvvm

7. MongoDB 컬렉션 명세
SensorData Collection
_id
ObjectId
MongoDB 자동 생성 고유 ID (Primary Key)
SensorId
String
센서 식별자 (예: "TEMP_001", "HUMID_002")
SensorType
String
센서 종류 (Temperature, Humidity, Power, Solar)
Value
Double
센서 측정값 (온도: ℃, 습도: %, 전력: kW, 태양광: kW)
Timestamp
DateTime
데이터 수집 시각 (UTC 기준, Index 적용)
FloorNumber
Int32
센서가 위치한 층 번호 (1, 2, 3)
Status
String
센서 상태 (Normal, Warning, Critical)

인덱스 설계

필드 타입 목적
Timestamp 단일 인덱스 시계열 데이터 조회 성능 향상 (최근 데이터 빠른 검색)
SensorId + Timestamp 복합 인덱스 특정 센서의 시간별 데이터 조회 최적화
FloorNumber 단일 인덱스 층별 센서 데이터 필터링 성능 향상
8. 기술적 고려사항

비동기 프로그래밍

async/await 패턴: MongoDB 데이터 저장/조회 시 UI 스레드 블로킹 방지
DispatcherTimer vs Timer: UI 업데이트는 DispatcherTimer 사용 (UI 스레드에서 직접 실행)

성능 최적화 전략

3D 렌더링: MeshBuilder로 30개 BoxVisual3D → 3개 통합 메시 (Draw Call 90% 감소)
Freezable 패턴: Brush 객체 Freeze()로 크로스 스레드 사용 및 메모리 최적화
더티 플래그: 색상 변경된 센서만 선택적 업데이트 (불필요한 렌더링 제거)

MVVM + DI 패턴

CommunityToolkit.Mvvm: Source Generator로 [ObservableProperty] 자동 생성
Microsoft.Extensions.DependencyInjection: MongoDBService 싱글톤 주입

예외 처리 및 안정성

MongoDB 연결 실패: 재연결 로직, 사용자에게 알림
센서 데이터 이상: 임계값 체크, Warning/Critical 상태 표시
메모리 누수 방지: Timer 정리, WeakEventManager 사용
