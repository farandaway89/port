1. 프로젝트 개요

프로젝트 목표

TensorFlow.js와 Face-API.js를 활용하여 웹 브라우저에서 실시간으로 얼굴 표정을 분석하고 감정을 인식하는 AI 시스템 개발. 웹캠을 통해 수집된 비디오 스트림에서 7가지 감정(행복, 슬픔, 화남, 두려움, 놀람, 혐오, 중립)을 분류하고, Chart.js를 통한 실시간 시각화 및 jsPDF를 활용한 임상 리포트 생성 기능을 제공합니다.

주요 기능

실시간 얼굴 표정 분석
웹캠 기반 실시간 얼굴 감지 및 표정 분류
7가지 감정 인식
딥러닝 모델을 통한 정확한 감정 분류
감정 변화 트래킹
Chart.js 기반 실시간 감정 변화 시각화
통계 분석
분석 시간, 감정 변화 횟수, 평균 신뢰도 제공
임상 리포트 생성
jsPDF를 활용한 전문적인 PDF 리포트 출력
데이터 내보내기
JSON, CSV 형식으로 분석 데이터 저장

7가지 감정 분류

Happy (행복) Sad (슬픔) Angry (화남) Fearful (두려움) Surprised (놀람) Disgusted (혐오) Neutral (중립)

기술 스택

JavaScript ES6+ TensorFlow.js 4.11.0 Face-API.js 0.22.2 Chart.js jsPDF HTML5 Canvas CSS3

AI 모델 구성

8. 기술적 고려사항

TensorFlow.js 모델 최적화

모델 로딩 전략: TinyFaceDetector 사용으로 모델 크기 최소화 (경량화), 초기 로딩 시간 단축
추론 최적화: requestAnimationFrame을 활용한 효율적인 프레임 처리, GPU 가속 활용 (WebGL 백엔드)
캐싱 전략: 모델 파일을 브라우저 캐시에 저장하여 재방문 시 빠른 로딩

실시간 처리 성능

프레임 스킵: 감지 간격 조정 (예: 100ms마다 처리)하여 CPU 부하 감소
Canvas 최적화: OffscreenCanvas 사용 고려, 필요한 영역만 다시 그리기
비동기 처리: async/await 패턴으로 UI 블로킹 방지, Web Workers 사용 검토

Chart.js 시각화

차트 업데이트 전략: addData/removeData 메서드로 부분 업데이트, 불필요한 리렌더링 방지
애니메이션 최적화: duration 조정으로 부드러운 전환 효과, 성능 모드 옵션 제공
데이터 포인트 제한: 타임라인 차트에서 일정 개수 이상 시 오래된 데이터 제거 (슬라이딩 윈도우)

브라우저 호환성

getUserMedia 폴백: 구형 브라우저를 위한 navigator.getUserMedia 폴리필
WebGL 지원: TensorFlow.js WebGL 백엔드 지원 여부 체크, CPU 백엔드 폴백
ES6+ 트랜스파일: Babel 사용으로 최신 JavaScript 기능 지원 (옵션)

개인정보 보호 및 보안

로컬 처리: 모든 비디오 및 감정 데이터는 브라우저 내에서만 처리, 서버 전송 없음
권한 관리: 카메라 권한 요청 시 명확한 안내 메시지, 사용자 동의 필수
데이터 삭제: 세션 종료 시 메모리에서 모든 데이터 자동 삭제, 명시적 초기화 기능 제공

에러 처리 및 예외 상황

카메라 미지원: getUserMedia 실패 시 사용자 친화적 오류 메시지 표시
모델 로딩 실패: 네트워크 오류 시 재시도 로직, CDN 장애 대비 로컬 모델 파일 백업
